# Changelog

All notable changes to prd-generator-plugin will be documented in this file.

## [2.1.0] - 2026-03-02

### Added
- New project skill: `project-docs` — generates and maintains a living `docs/` documentation tree at the project root
- `docs/index.md` as the project documentation entry point, with table of contents linking to 6 standard sub-files
- Standard doc sub-files: `docs/business.md`, `docs/architecture.md`, `docs/structure.md`, `docs/development.md`, `docs/tech-stack.md`, `docs/local-setup.md`
- Mermaid diagram support in generated docs with accessible color palette for light/dark backgrounds
- API key placeholder enforcement in `docs/local-setup.md` (never real values)
- `/prd-docs` slash command for manual documentation refresh
- `docs/` root files added to `/prd-new` Phase 5 (CREATE) and Phase 6 verification checklist
- `project-docs` skill always included in `/prd-evolve` affected skills dispatch
- `project-docs` added as item 5 in `prd-guardian` required check order

### Changed
- Plugin generates 7 project skills per project (was 6)
- `/prd-new` Phase 5 now includes a sequential docs CREATE step after the 4 parallel agents
- `/prd-new` Phase 7 commit now includes `docs/*.md` files
- `/prd-new` completion summary reports 7 skills and lists generated docs files
- `prd-guardian` quick reference table updated to include `project-docs` and `docs/index.md`
- README monorepo structure diagram updated to show `docs/` root files

## [2.0.0] - 2026-03-01

### Added
- Monorepo as mandatory project structure for all generated projects — enforced by `project-guardian` Hard Block
- `/prd-new` Block 6 now asks VCS (GitHub/GitLab/Bitbucket) and branch strategy (main-only/gitflow/trunk-based)
- New agent: `cicd-generator` — generates lean, path-filtered CI/CD pipelines for GitHub Actions, GitLab CI, and Bitbucket Pipelines
- New project skill: `project-docs-stack` — enforces local-first documentation cache in `docs/stack/`
- New project skill: `project-cicd` — tracks pipeline consistency and triggers evolution on stack changes
- `docs/stack/README.md` index file generated during `/prd-new`
- `.claude/settings.json` with `PreToolUse` hook for `WebFetch`/`WebSearch` generated during `/prd-new`
- `/prd-evolve` now re-generates CI/CD pipeline when `delta_type` includes tech or architecture changes
- `repo_structure: "monorepo"` hardcoded in `context_packet` — not a user choice

### Changed
- **Interrogation format**: Questions now delivered one-at-a-time as plain chat output (markdown), not as multi-question `AskUserQuestion` forms — improves rendering reliability
- Phase 5 of `/prd-new` now dispatches 4 parallel agents (added `cicd-generator`)
- `project-guardian` skill template now includes Hard Block for non-monorepo repository creation
- Git commits now include CI/CD files, `docs/stack/`, and `.claude/settings.json`
- Plugin generates 6 project skills per project (was 4)
- `stack-guide-generator` now generates `docs/stack/` directory and `.claude/settings.json` hook in addition to CLAUDE.md files
- Root CLAUDE.md generated by `stack-guide-generator` now includes Monorepo Structure and Docs Policy sections
- `architecture-designer` now includes monorepo constraint in design philosophy and output format
- Completion summary reports 6 skills (was 4)

## [1.0.0] - 2026-02-28

### Added
- `/prd-new` command — full interactive PRD generation workflow
- `/prd-evolve` command — incremental PRD evolution with selective artifact updates
- 8 specialized sub-agents: product-interrogator, official-researcher, research-validator, requirements-analyst, architecture-designer, prd-writer, stack-guide-generator, skills-generator
- `prd-guardian` global skill for PRD compliance enforcement
- Official-sources-only research with blocked_domains enforcement
- Fresh-context research validation loop (max 3 iterations)
- Modern stack selection optimized for AI-assisted development
- Stack-specific CLAUDE.md guides based on Clean Code, Clean Architecture, DDD, The Pragmatic Programmer
- 4 self-evolving project enforcement skills: project-guardian, project-architecture, project-domain-rules, project-compliance
- Stale skills version detection
- Token-efficient JSON context_packet handoff pattern
- Edge case handling: no-git-repo, early interrogation cancel, research failure fallback, missing PRD guard
